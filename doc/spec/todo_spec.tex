\section*{Anforderungen des Kunden}
Anbei die Anforderungen des Kunden gemäß der Email vom 30.01.2003

\begin{itemize}

\item Form: DIN A4, genügend Platz für Korrektur und handschriftliche 
Anmerkungen


\item Gedruckte Dokumentation + PDF

\item Versionierung: Jedes Dokument erhält eine Revisionsnummer, 
Erstellungsdatum wird vermerkt

\item Inhaltliche Form: Jedes Dokument verfügt über ein Inhaltsverzeichnis,
Index ist erwünscht, Seitenzahlen


\item Referenzen: Konsistente globale Nummerierung für alle Dokumente
(lebenslang und für immer und ewig)

\item Querverweise (innerhalb des selben Dokuments), 
Referenzen auf weitere Dokumente über Seite + Abschnitt

\item Inhalt: Begriffslexikon, Konzepte der GUI, 
Erklärungen der GUI zum Beispiel anhand abstrakter Bilder,
Usecases mit Zielsetzung spezifizieren

\item vollständige Abdeckung der Anforderungen an das Produkt

\item präzise, knapp

\item Traceability: \\
Spezifikation ist der erste Schritt auf dem langen
(und beschwerlichen) Weg zum fertigen Software-Produkt und
dient als Grundlage für das weitere Vorgehen.\\
Entwurf, Implementierung und Test referenzieren sie.
Testfälle für den Systemtest sollen aus der Spezifikation
ableitbar sein.


\item Sprache:\\
Wenn Deutsch als Sprache gewählt wird, sollten englische Fach-Begriffe
verwendet werden. (Die selben Begriffe werden in Spezifikation
und Implementierung verwendet.)\\
Keine Eindeutschung.\\
Einheitliche Rechtschreibung.

\end{itemize}
